# 🛰️ Simulation Framework for *"A Comparative Study of Game-Theoretic and Reinforcement Learning Approaches for Jam-Resilient Wireless Networks in IoT"*

[cite_start]
This repository provides the complete Python simulation framework used for the research paper:  
**"A Comparative Study of Game-Theoretic and Reinforcement Learning Approaches for Jam-Resilient Wireless Networks in IoT"** [cite: 63, 436].

The framework is designed to conduct a **round-robin tournament**, pitting various attacker and defender models against each other in a simulated wireless IoT network environment. It evaluates adaptive defense strategies—specifically **Bayesian game-theoretic models** and **Q-Learning reinforcement learning agents**—against a suite of adaptive jamming attacks.

The simulator is built to be **modular**, allowing researchers to easily test new agent models, strategies, and network topologies.

---

## ⚙️ Features

* **Agent Models:** Implements three core agent types for both attackers and defenders:
  * 🧠 **Q-Learning:** A reinforcement learning agent that learns an optimal policy from network state observations.
  * 🎲 **Bayesian Game:** A game-theoretic agent that maintains probabilistic beliefs over its opponent's strategies and chooses the action with the highest expected utility.
  * ⚡ **Static:** A baseline agent that follows a predefined (e.g., random or fixed) policy.

* **Modular Strategies:** Defines a comprehensive set of discrete strategies for both players:
  * **Attacker Strategies:** `broadband`, `sweep`, `reactive`, `targeted`, `power_burst`, `intelligent`, `reactive_strong`, `central_node_targeting`
  * **Defender Strategies:** `hop`, `detect_and_switch`, `stay`, `spread_spectrum`, `error_coding`, `cooperative`

* **Network Topologies:** Generates various network graphs via `networkx` to simulate different IoT deployment scenarios, including **Erdős-Rényi**, **Star**, **Ring**, and **Small-World** topologies.

* **Comprehensive Metrics:** Logs key performance indicators such as attacker/defender payoffs, network health, jammer detection rates, and strategy usage frequencies.

* **Automated Plotting:** Includes `plot_results.py`, which parses simulation outputs and automatically generates all plots (heatmaps, convergence charts, boxplots, etc.) presented in the paper.

## 🏃 Running the Simulation

Running the full analysis involves two main steps:  
**(1)** Running experiments to generate raw data, and  
**(2)** Plotting results for analysis.

---

### Step 1: Run the Experiments

Execute the following command:
```bash
python run_experiments.py
```

**This will:**
* Iterate through all combinations of attacker/defender models and network topologies.
* Create a **timestamped results directory** (e.g., `results_20251110-150000/`).
* Run multiple independent trials (`RUNS_PER_SETTING`) for each configuration.
* Save two types of data:
  1. **Interval Logs (`*_interval_log.csv`)** – Aggregated metrics every few steps (e.g., 50).
  2. **Detailed Histories (`detailed_histories_consolidated/`)** – Step-by-step logs for learning models.

> ⚠️ **Note:** This step can be computationally intensive depending on your hardware and number of trials.

---

### Step 2: Generate Plots

Once simulations finish, run:
```bash
python plot_results.py
```

**This script will:**
* Automatically locate the **most recent results directory**.
* Aggregate and analyze all CSV logs.
* Generate all figures (PDF format) used in the paper.
* Save plots in:  
  `results_YYYYMMDD-HHMMSS/plots_main_paper/`

---

### Step 3: View Results

* **Raw Data:** `results_YYYYMMDD-HHMMSS/`  
* **Generated Figures:** `results_YYYYMMDD-HHMMSS/plots_main_paper/`

## 🔬 Simulation Overview & Customization

This framework is designed to be **configurable and extendable**.

### 🔧 Customizing Experiments

Modify configuration variables inside `run_experiments.py` under:
```python
if __name__ == "__main__":
```

Key parameters include:

| Variable | Description |
|-----------|-------------|
| `ATTACKER_MODELS` | List of attacker models to test |
| `DEFENDER_MODELS` | List of defender models to test |
| `TOPOLOGIES` | Network topologies (e.g., Ring, Star) |
| `NODE_COUNTS` | Network sizes (e.g., `[10, 20, 50]`) |
| `RUNS_PER_SETTING` | Number of independent trials |
| `SIMULATION_STEPS` | Duration (in steps) per run |
| `Q_LEARNING_PARAMS_GRID` | Hyperparameters for Q-Learning (α, γ, ε) |

---

### 🧩 Code Engine: `core_sim.py`

Defines the simulation's logic and agent intelligence.

* `generate_network`: Creates a network topology via `networkx`.
* `execute_attack` / `execute_defense`: Implement jamming and counter-jamming logic.
* `payoff`: Calculates attacker/defender rewards based on network conditions.
* `get_q_learning_state`: Encodes state representation for RL agents.
* `select_attacker_strategy` / `select_defender_strategy`: Decision-making logic (e.g., ε-greedy, expected utility).
* `update_learning_models`: Updates Q-tables or Bayesian beliefs after each step.

---

### 🧠 Experiment Orchestrator: `run_experiments.py`

Coordinates the end-to-end experiment workflow.

Each simulation:
1. Initializes agents and network topology.  
2. Loops for `SIMULATION_STEPS`, performing:
   * State observation
   * Strategy selection
   * Action execution
   * Payoff computation
   * Learning updates
   * Metric logging  
3. Saves results to structured output files for later analysis.

---

## 📊 Post-Processing: `plot_results.py`

* Aggregates interval logs and detailed histories.  
* Generates publication-quality visualizations:
  * Heatmaps  
  * Convergence curves  
  * Boxplots  
  * Comparative payoff distributions  

All figures are exported in PDF format for inclusion in LaTeX papers.
